# scrcpy v4.0 GUI ランチャー アプリケーション 要求定義書

**プロジェクト名：** scrcpy GUI Launcher  
**バージョン：** 1.0  
**作成日：** 2026年5月17日  
**対象バージョン：** scrcpy v4.0

---

## 1. プロジェクト概要

### 1.1 背景と目的

scrcpy はコマンドラインツールであり、多数のオプションを手動で記述する必要があるため、技術に不慣れなユーザーにとって利用ハードルが高い。本アプリケーションは、scrcpy v4.0 の主要オプションをGUI上で選択・設定できる「ランチャー」として機能し、コマンドライン操作なしで scrcpy を実行できる環境を提供することを目的とする。

### 1.2 対象ユーザー

| ユーザー種別 | 説明 |
|---|---|
| 一般ユーザー | コマンドライン操作に不慣れで、スマートフォンをPCで操作したい人 |
| エンジニア・開発者 | Androidアプリ開発やデバッグ用途でscrcpyを活用したい人 |
| VRユーザー | Meta QuestをPCへミラーリングしたい人 |
| 配信者・クリエイター | ゲームやスマホ画面を録画・配信したい人 |

### 1.3 スコープ

- **対象OS：** Windows 10 / Windows 11（64bit）
- **前提：** scrcpy v4.0 の実行ファイル（`scrcpy.exe`）が同一フォルダまたは指定パスに存在すること
- **プラットフォーム：** デスクトップWebアプリ（HTML / CSS / JavaScript のシングルファイル）または Electron アプリ
- **対象外：** scrcpy 本体のインストール機能、macOS / Linux 対応

## 2. システム構成

### 2.1 アーキテクチャ概要

```
[GUI Launcher (HTML/JS or Electron)]
         │
         │  コマンド生成
         ▼
[scrcpy.exe 実行ファイル]
         │
         │  ADB通信
         ▼
[接続済みAndroidデバイス / Meta Quest]
```

### 2.2 技術スタック候補

| 方式 | 技術 | 備考 |
|---|---|---|
| A案（推奨） | Electron + HTML/CSS/JS | デスクトップアプリとして配布可能。ADB・scrcpyの実行が容易 |
| B案 | Python + Tkinter or PyQt | 軽量。配布はexe化（PyInstaller）が必要 |
| C案 | WPF / WinForms (.NET) | Windowsネイティブ。UI構築コストが高い |

### 3.2 機能詳細

#### F-001：デバイス検出・選択

- 「デバイスを更新」ボタン押下で `adb devices` を実行し、接続済みデバイスを一覧表示する
- 接続デバイスが0台の場合：「デバイスが見つかりません。USBを接続してUSBデバッグを許可してください」と表示
- 接続デバイスが1台の場合：自動選択
- 接続デバイスが2台以上の場合：ドロップダウンで選択（デバイス名・シリアルIDを表示）
- デバイスの状態（`device` / `offline` / `unauthorized`）をアイコンで視覚的に表示
- `offline` 状態の場合：トラブルシューティング手順をポップアップで案内

#### F-002：接続方式の選択

- USB接続 / Wi-Fi接続（ワイヤレスデバッグ）を切り替えるタブまたはラジオボタン
- **Wi-Fi接続の場合：**
  - IPアドレス入力フィールド（例：`192.168.1.xx`）
  - ポート番号入力フィールド（デフォルト：`5555`）
  - 「接続」ボタンで `adb connect IP:PORT` を実行
  - ワイヤレスデバッグ（Android 11+）のペアリング支援機能：ペアリングコード入力フォームと `adb pair` 実行ボタン

#### F-003：基本表示設定

| 設定項目 | UI部品 | 対応オプション | デフォルト値 |
|---|---|---|---|
| 最大解像度 | スライダー or 数値入力 | `--max-size` | 0（制限なし） |
| 最大フレームレート | スライダー | `--max-fps` | 60 |
| ビットレート | スライダー or 数値入力 | `--video-bit-rate` | 8M |
| フルスクリーン起動 | トグルスイッチ | `--fullscreen` | OFF |
| 常に最前面 | トグルスイッチ | `--always-on-top` | OFF |
| 画面回転 | セレクトボックス（0°/90°/180°/270°） | `--rotation` | 0（変更なし） |
| 音声なし | トグルスイッチ | `--no-audio` | OFF |
| スリープ防止 | トグルスイッチ | `--keep-active` | OFF |
| 終了時ADB停止 | トグルスイッチ | `--kill-adb-on-close` | OFF |

#### F-004：仮想ディスプレイ設定（scrcpy v4.0 新機能）

- 仮想ディスプレイ機能の有効/無効をトグルで切り替え
- **フレックスディスプレイ（`-x`）：** トグルスイッチ。ONの場合は `--new-display` と自動的に組み合わせる
  - フレックスディスプレイON時はサイズ入力を無効化し、DPIのみ入力可能にする
- **ディスプレイ解像度：** 幅×高さの数値入力（例：`1280x720`）。フレックスON時は非表示
- **DPI：** 数値入力（デフォルト：160）
- **終了時コンテンツ維持：** トグル（`--no-vd-destroy-content`）
- **起動アプリ指定：** テキスト入力（例：`com.android.settings`）。`--start-app` に対応
- **システムデコレーション非表示：** トグル（`--no-vd-system-decorations`）

#### F-005：カメラ設定（scrcpy v4.0 新機能）

- カメラミラーリングモードの有効/無効をトグルで切り替え
- **カメラ選択：** フロント / リア のセレクトボックス（`--camera-facing`）
- **初期ズーム：** スライダー（`--camera-zoom`）
- **トーチ（ライト）：** トグルスイッチ（`--camera-torch`）
- カメラ設定説明文：「カメラモードON時は通常のスクリーンミラーリングではなくカメラ映像を表示します」

#### F-006：録画設定

- 録画の有効/無効をトグルで切り替え
- **出力ファイルパス：** テキスト入力 + ファイルダイアログボタン（`--record`）
- **出力フォーマット：** セレクトボックス（MP4 / MKV / H264 / OPUS）
- **録画中ミラーリング非表示：** トグル（`--no-display`）
- デフォルトファイル名：`scrcpy_YYYYMMDD_HHMMSS.mp4`（日時を自動付与）

#### F-007：コマンドプレビュー

- 画面下部に現在の設定から生成される実行コマンドをリアルタイムで表示する
  ```
  scrcpy -s [device] --new-display -x --new-display=/192 --keep-active
  ```
- 「コピー」ボタンでクリップボードにコピー可能
- コマンドは設定変更のたびに即座に更新される

#### F-008：プリセット保存・読み込み

- 現在の設定をプリセットとして名前付きで保存できる
- 保存したプリセットをドロップダウンから呼び出して設定を復元できる
- プリセット例：「開発用・仮想ディスプレイ」「録画用・高画質」「Quest3ミラーリング」
- プリセットはJSONファイルとしてローカルに保存（ファイルパスはユーザー指定可）

#### F-009：実行ログ表示

- scrcpy の起動・実行・終了ログをアプリ内のテキストエリアに表示する
- INFOメッセージは通常色、ERRORメッセージは赤色で表示
- 「ログをクリア」ボタン
- 「ログをコピー」ボタン

#### F-010：scrcpy 実行パス設定

- scrcpy.exe の実行ファイルパスをUIから設定できる
- デフォルトは同一フォルダ内の `scrcpy.exe` を自動検索
- 「参照」ボタンでファイルダイアログから選択可能
- パスが無効な場合は警告を表示し、「実行」ボタンを無効化する

```

│  ─────────────  │  │ 表示 │ 仮想画面 │ カメラ│録画 │ │  ← メインコンテンツ
│  [デバイスA ▼]  │  ├──────────────────────────────┤ │
│  [🔄 更新]      │  │                               │ │
│                  │  │  各タブの設定項目              │ │
│  🔌 接続方式    │  │                               │ │
```

### 4.2 デザイン要件

- **ダークモード / ライトモード：** 両対応。システム設定に従いデフォルト切り替え、手動トグルも提供
- **配色：** テック系・ダークテーマを基調。アクセントカラーは緑系（Androidブランドに合わせて）
- **フォント：** 日本語表示対応（`Noto Sans JP` または `Yu Gothic UI`）
- **レスポンシブ：** 最小ウィンドウサイズ 960px × 600px
- **アイコン：** Lucide Icons または Material Icons を使用
- **言語：** 日本語UI（将来的に多言語対応を考慮した設計）

### 4.3 UX 要件

- 設定変更時にコマンドプレビューがリアルタイム更新されることで、ユーザーが生成されるコマンドを常に確認できる
- 初回起動時にオンボーディングガイド（3ステップのチュートリアル）を表示する
- エラー発生時（ADBエラー・デバイス未接続など）は原因と解決策をわかりやすく案内する
- ツールチップで各オプションの説明を表示する（特に技術的なオプション）
- 「実行」ボタンはデバイスが未選択または scrcpy パスが無効の場合は無効化する

## 5. 非機能要求

### 5.1 パフォーマンス

- アプリ起動時間：3秒以内
- デバイス検出（`adb devices` 実行）：5秒以内にレスポンス
- コマンドプレビューの更新：設定変更から100ms以内に反映

### 5.2 信頼性・安全性

- scrcpy や ADB のプロセスがクラッシュしてもランチャー自体はクラッシュしない（例外処理必須）
- ユーザーが入力したパスに存在しないファイルを指定した場合は起動前に検証・エラー表示する
- scrcpy 実行中に「停止」ボタンで安全にプロセスを終了できる

### 5.3 配布・インストール

- インストーラー不要。ZIPを展開して実行するポータブル形式を推奨
- scrcpy v4.0 の実行ファイル群と同梱可能な構成にする
- Electron 採用の場合は `electron-builder` でWindowsインストーラー（NSIS）も提供

## 8. 制約事項・前提条件

- scrcpy v4.0 の実行ファイルが別途必要（本アプリはランチャーであり、scrcpy本体は含まない）
- ADB（Android Debug Bridge）が `scrcpy.exe` と同梱されているため、別途インストールは不要
- 接続対象のAndroidデバイスは「USBデバッグ」または「ワイヤレスデバッグ」が事前に有効化されていること
- フレックスディスプレイ（`-x`）は `--new-display` との組み合わせでのみ動作する
- Meta Quest への接続は通常のAndroidデバイスと同様の手順で対応可能

## 10. 用語定義

| 用語 | 説明 |
|---|---|
| scrcpy | Genymobile製のオープンソースAndroidミラーリングツール |
